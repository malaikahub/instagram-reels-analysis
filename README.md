## 📝 Project Description

This project extracts Reels data from the Instagram account `@ali_raza943` in two different ways — one where Selenium auto-scrolls the reels page to collect links, and another where 20 reel links are hardcoded and their data is fetched directly via Apify. Work started locally in VS Code on a Windows machine, where three different scraping methods (Instaloader, Selenium, BeautifulSoup) were tried and each failed with a different error. After exhausting all local options, the final, working pipeline moved to **Google Colab + Apify**.

## 🔄 Methods Tried in VS Code (and Why They Failed)

All three methods below were implemented and run locally inside VS Code on a Windows machine. Each one failed with a different error, and multiple fixes were attempted for each before moving on.

### ❌ Method 1 — Instaloader
Instaloader connects directly to Instagram's internal API using a username/password, without needing a browser.

- **Error:** `fail — CSRF token missing or incorrect`, thrown immediately on login.
- **Why it failed:** Instagram requires a security token that is only generated by a real browser session. Instaloader skips the browser entirely, so the login is blocked before it even starts.
- **Fixes tried:** adding `time.sleep()` delays, saving/reloading session cookies, changing the User-Agent string — none worked; Instagram blocked the attempt at the IP level.

### ❌ Method 2 — Selenium
Selenium opens a real Chrome window and controls it like a human (clicking, typing, scrolling) to log in, navigate to the Reels tab, auto-scroll, and collect data.

- **Errors:**
  - `TimeoutException` — the username field was never found because the page loaded blank or a cookie-consent popup blocked the login form.
  - Instagram sometimes triggered a verification/unusual-activity challenge that blocked the script.
  - Views, likes, and comment counts returned `N/A` even though the numbers were visible on screen — extraction selectors didn't match Instagram's current page structure.
  - Auto-scroll stopped too early, before collecting all 20 reel links.
- **Fixes tried:** switched the target URL to `/accounts/login/` directly, increased wait time from 5s to 10s, added a `close_popups()` function for cookie banners, injected CDP JavaScript to hide the automation flag, added multiple fallback selectors plus a full-page body-text scan for likes/views/comments — results improved but stayed unreliable, and the login form still failed intermittently.

### ❌ Method 3 — Requests + BeautifulSoup
A simple script sent an HTTP request directly to the Reels page and tried to parse reel links/data from the raw HTML, with no browser involved.

- **Error:** No exception was thrown, but the result was completely empty — no reel links or data found.
- **Why it failed:** Instagram is built with React.js. The browser first loads an empty HTML shell, then JavaScript fetches and renders the real content. BeautifulSoup has no JavaScript engine, so it only ever sees the empty shell.
- **Fixes tried:** added realistic browser headers (`Accept-Language`, `Referer`) and injected cookies from a logged-in session — the HTML response stayed empty every time, since JavaScript never actually ran.

After all three methods failed despite multiple fix attempts, it became clear the local-machine approach had fundamental limitations against Instagram's bot detection and JavaScript rendering.

---

## ✅ Final Solution — Google Colab + Apify

Apify is a cloud-based scraping platform: the scraper runs on Apify's own servers using their own IPs, so Instagram doesn't block it. No Selenium, no browser installation, and no login credentials are needed.

**Problems faced during Apify setup, and how they were solved:**

| Problem | Solution |
|---|---|
| Authentication error (wrong API token) | Created a free account at apify.com → Settings → Integrations → copied the correct Personal API Token |
| "Actor not found" (`instagram/scraper` doesn't exist) | Used the correct actor name with a dash: `apify/instagram-scraper` |
| Free plan returned very few/zero comments per reel | Reduced `MAX_REELS` to a smaller batch size to stay within free-tier limits |
| Comments returned as a string instead of a list (`TypeError` on loop) | Added a type check using `ast.literal_eval()` / `json.loads()` to safely convert the string to a list |
| Inconsistent column names across runs (`videoViewCount` vs `videoPlayCount`) | Used `row.get()` with a fallback value for every field |

Once these issues were resolved, the scraper successfully fetched reel data for `@ali_raza943` — including views, likes, comment counts, captions, and post dates — and the results were saved as CSV files and downloaded directly from Colab.

Two parallel approaches were kept for comparison once the Apify pipeline worked:
- **`autoscroll.py`** — Selenium auto-scrolls the live reels page in Chrome to *discover* reel links, then extracts data for 20 reels.
- **`hardcoded.py`** — Uses a fixed, pre-collected list of 20 reel links and pulls their data directly via Apify.
